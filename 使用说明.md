## 项目介绍

Claude Code v2.1.88 是 Anthropic 开发的一款强大的 AI 编码助手，它能够：

- 理解自然语言指令并生成代码
- 提供智能代码补全和建议
- 执行工具操作（如文件操作、bash 命令等）
- 支持多种模型切换
- 提供交互式 REPL 环境

## 安装步骤

### 前提条件

- Node.js >= 18.0.0
- npm 或 yarn 包管理器

### 安装方法

1. **克隆仓库**

```bash
git clone <仓库地址>
cd <项目目录>
```

2. **安装依赖**

```bash
npm install
```

3. **构建项目**

```bash
npm run build
```

4. **运行项目**

```bash
node dist/cli.js
```

## 功能介绍

### 核心功能

1. **代码生成** - 根据自然语言描述生成代码

## 操作指南

### 基本操作

1. **启动应用**

```bash
node dist/cli.js
```

2. **输入指令**

在 REPL 环境中，直接输入您的指令，例如：

```
> 帮我写一个快速排序算法
```

3. **使用工具**

使用斜杠命令调用工具：

```
> /bash ls -la
```

4. **查看帮助**

```
> /help
```

### 高级操作

1. **设置模型**

使用 `--model` 参数指定模型：

```bash
node dist/cli.js --model sonnet
```

2. **启用快速模式**

```bash
node dist/cli.js --fast
```

3. **会话管理**

```
> /session save my-session
> /session load my-session
```

## 模型切换功能

Claude Code 支持在不同的 Claude 模型之间切换，以满足不同的需求。

### 可用模型

- **Opus** - 最强大的模型，适合复杂任务
- **Sonnet** - 平衡性能和速度，适合大多数任务
- **Haiku** - 轻量级模型，速度快，适合简单任务

### 如何切换模型

1. **使用命令行参数**

```bash
node dist/cli.js --model sonnet
```

2. **在 REPL 中使用模型选择器**

```
> /model
```

这将打开模型选择界面，您可以使用上下箭头选择模型，按 Enter 确认。

3. **设置默认模型**

在 `~/.claude/settings.json` 文件中设置默认模型：

```json
{
  "model": "sonnet"
}
```

### 模型特性对比

| 模型 | 性能 | 速度 | 适合场景 |
|------|------|------|----------|
| Opus | 最高 | 较慢 | 复杂编码、深度分析 |
| Sonnet | 高 | 中等 | 一般编码任务、日常使用 |
| Haiku | 中等 | 最快 | 简单任务、快速原型 |

## 常见问题解答

### 1. 构建失败怎么办？

**问题**：运行 `npm run build` 时失败，出现模块缺失错误。

**解决方案**：
- 确保所有依赖已正确安装：`npm install`
- 检查是否缺少必要的 stub 文件
- 查看构建日志，了解具体错误信息

### 2. 模型切换不生效

**问题**：使用 `--model` 参数或 `/model` 命令切换模型后，模型没有变化。

**解决方案**：
- 检查模型名称是否正确（opus、sonnet、haiku）
- 确保设置文件没有覆盖命令行参数
- 重启应用后再次尝试

### 3. 工具执行失败

**问题**：执行 bash 命令或其他工具时失败。

**解决方案**：
- 检查权限设置
- 确保命令语法正确
- 查看错误信息了解具体原因

### 4. 应用启动缓慢

**问题**：应用启动时间较长。

**解决方案**：
- 确保 Node.js 版本 >= 18.0.0
- 检查系统资源使用情况
- 考虑使用快速模式：`--fast`

### 5. 如何更新模型？

**问题**：如何获取最新的模型版本？

**解决方案**：
- 定期更新 Claude Code 到最新版本
- 模型更新由 Anthropic 服务器端管理，无需本地更新

### 6. 如何配置 API 密钥？

**问题**：需要配置 API 密钥才能使用。

**解决方案**：
- 在首次运行时，应用会提示您输入 API 密钥
- 或者在 `~/.claude/settings.json` 文件中设置：

```json
{
  "apiKey": "your-api-key-here"
}
```

### 7. 如何使用自定义模型？

**问题**：如何使用自定义模型配置？

**解决方案**：
- 在 `~/.claude/settings.json` 文件中设置 `modelOverrides`：

```json
{
  "modelOverrides": {
    "claude-opus-4-6": "your-custom-model-id"
  }
}
```

### 8. 应用崩溃怎么办？

**问题**：应用突然崩溃或无响应。

**解决方案**：
- 检查系统日志获取错误信息
- 尝试重启应用
- 如果问题持续，考虑重新安装

## 故障排除

### 查看日志

应用日志存储在 `~/.claude/logs/` 目录中，您可以查看这些日志了解详细的错误信息。

### 重置设置

如果遇到配置问题，可以重置设置：

```bash
rm -rf ~/.claude/settings.json
```

然后重新启动应用，按照提示重新配置。
